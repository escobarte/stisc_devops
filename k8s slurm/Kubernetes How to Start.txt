В Kubernetes (и вообще в DevOps) слово “манифест” взято из английского “manifest”, что значит “заявление”, “декларация”.
Это исторически термин из мира логистики: манифест — это документ, где перечислено, что находится на корабле, поезде, самолёте (перечень грузов, пассажиров).
Почему так назвали?
В Kubernetes манифест — это файл, в котором декларируется (“заявляется”) желаемое состояние объекта (например, “хочу, чтобы у меня был вот такой Pod с такими-то контейнерами, вот такими томами и портами”).
Манифест как бы “заявляет системе”, что должно быть.


====================================
      KUBERNETES HOW TO START
====================================

Если хочешь просто поиграться с Kubernetes локально:
---------------------------------
````💡[1] MINIKUBE
---------------------------------
Для одной машины/ноутбука — самый быстрый способ.
Разворачивает кластер в одной виртуалке (или через Docker).
Подходит для тестов, изучения kubectl, деплойментов и т.д.

---------------------------------
````💡[2] Kind (Kubernetes IN Docker)
---------------------------------
Создаёт кластер прямо в Docker-контейнерах.
Удобно для быстрого старта, CI/CD, локальной разработки

Если хочешь получить опыт, максимально похожий на продакшн:
---------------------------------
````💡[3] kubeadm
---------------------------------
На нескольких виртуалках/серваках (можно в VirtualBox, VMware или в облаке).
Нужно минимум две машины (master + worker), можно на одной для учебы.
Ты сам настраиваешь сеть, устанавливаешь компоненты — зато реально учишься!
========================================================
УСТАНОВКа "MINIKUBE"
========================================================
[STEP 1] Установи Docker
[STEP 2] Установи kubectl
https://v1-32.docs.kubernetes.io/docs/tasks/tools/install-kubectl-linux/
[STEP 3] Установи Minikube
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube
[STEP 4] Запусти Minikube
minikube start --driver=docker
 [NOTE]Если будет ошибка прав:
 - выйди из рута
 — запусти sudo usermod -aG docker $USER и перелогинься
 - groups $USER #прорь
[STEP 4.1]Проверь
kubectl get nodes
